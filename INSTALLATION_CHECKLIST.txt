## üì¶ LOCAL SETUP CHECKLIST (Windows/Mac/Linux)

### Step 1: Prepare Project
- [ ] Extract `gram_panchayat.zip` file
- [ ] Open terminal/command prompt
- [ ] Navigate to project directory: `cd gram_panchayat`

### Step 2: Virtual Environment
```
Windows:
- [ ] python -m venv venv
- [ ] venv\Scripts\activate

Mac/Linux:
- [ ] python3 -m venv venv
- [ ] source venv/bin/activate
```

### Step 3: Install Dependencies
- [ ] Run: `pip install -r requirements.txt`
- [ ] Wait for installation to complete (2-5 minutes)
- [ ] Verify: `pip list` shows Django 4.2

### Step 4: Configure Environment
- [ ] Copy .env.example to .env: `cp .env.example .env`
- [ ] Edit .env file with your settings
- [ ] Set DEBUG=True for development
- [ ] Generate SECRET_KEY from https://djecrety.ir/
- [ ] Paste SECRET_KEY into .env

### Step 5: Initialize Database
- [ ] Run: `python manage.py migrate`
- [ ] Verify: db.sqlite3 file created
- [ ] Run: `python manage.py createsuperuser`
- [ ] Enter admin username (e.g., "admin")
- [ ] Enter admin email (e.g., "admin@fungao.com")
- [ ] Enter password twice

### Step 6: Collect Static Files
- [ ] Run: `python manage.py collectstatic --noinput`
- [ ] Verify: staticfiles/ folder created

### Step 7: Test Locally
- [ ] Run: `python manage.py runserver`
- [ ] Open browser to: http://127.0.0.1:8000
- [ ] [ ] Homepage loads successfully
- [ ] [ ] All pages are accessible
- [ ] [ ] Admin panel at http://127.0.0.1:8000/admin/ works
- [ ] [ ] Login with your admin credentials

### Step 8: Add Sample Data
- [ ] Go to Django admin: `/admin/`
- [ ] [ ] Add village information
- [ ] [ ] Add panchayat members with photos
- [ ] [ ] Add 2-3 government schemes
- [ ] [ ] Upload 5-10 photos to gallery
- [ ] [ ] Send test contact message

### Step 9: Test Admin Features
- [ ] [ ] Edit village information
- [ ] [ ] Update member details
- [ ] [ ] Add new scheme
- [ ] [ ] Upload gallery image
- [ ] [ ] View contact messages
- [ ] [ ] Everything works as expected

## üåê DEPLOYMENT CHECKLIST (Choose One)

### Option A: RENDER.COM (Recommended)

#### Pre-Deployment
- [ ] Create GitHub account (if not already)
- [ ] Push project to GitHub repository
- [ ] Ensure .gitignore prevents .env upload

#### Deployment Steps
- [ ] Create account at render.com
- [ ] Connect GitHub account to Render
- [ ] Click "New" ‚Üí "Web Service"
- [ ] Select your gram-panchayat repository
- [ ] Configure settings:
  - [ ] Name: gram-panchayat
  - [ ] Environment: Python 3
  - [ ] Build command: `pip install -r requirements.txt && python manage.py collectstatic --noinput`
  - [ ] Start command: `gunicorn gram_panchayat.wsgi:application`

#### Environment Variables
- [ ] Add SECRET_KEY variable
- [ ] Add DEBUG=False
- [ ] Add ALLOWED_HOSTS=yourdomain.onrender.com
- [ ] Generate and add other required vars

#### Database
- [ ] Create PostgreSQL database on Render
- [ ] Copy DATABASE_URL to web service vars
- [ ] Click Deploy

#### Post-Deployment
- [ ] Wait 2-5 minutes for deployment
- [ ] [ ] Visit deployed URL and verify it works
- [ ] [ ] Admin panel accessible
- [ ] [ ] Database migrated automatically
- [ ] [ ] Create superuser on production
- [ ] [ ] Test all features

---

### Option B: RAILWAY.APP

#### Setup
- [ ] Install Railway CLI: `npm i -g @railway/cli`
- [ ] Login: `railway login`
- [ ] In project folder: `railway init`
- [ ] [ ] Select gram-panchayat project

#### Environment
- [ ] Run: `railway variables set SECRET_KEY "your-key"`
- [ ] Run: `railway variables set DEBUG "False"`
- [ ] Run: `railway variables set ALLOWED_HOSTS "yourdomain.railway.app"`

#### Database
- [ ] Run: `railway add` and select PostgreSQL
- [ ] [ ] Railway auto-creates DATABASE_URL

#### Deployment
- [ ] Run: `railway up`
- [ ] [ ] Wait for deployment complete
- [ ] [ ] Visit your railway.app URL

---

### Option C: PYTHONANYWHERE.COM

#### Account Setup
- [ ] Create account at pythonanywhere.com
- [ ] Verify email

- [ ] [ ] Copy WSGI template from docs
- [ ] [ ] Update path to /home/yourusername/gram_panchayat/

#### Files & Paths

- [ ] Set media files (if needed)

#### Database

- [ ] [ ] Create database

#### Launch
